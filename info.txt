Sí. Como ya fueron distribuidos, lo correcto es “retroceder” esos documentos específicos y volver a procesarlos, no limpiar todo el mes.
Hazlo así:
1. Identifica los IDs afectados
Por ejemplo:
SQL
SELECT id, asiento_contable, archivo_fuente, tipo_detectado, estado
FROM documentos_paginas
WHERE cliente_abreviatura='CIMA'
  AND anio=2026
  AND mes=1
  AND id IN (2063, 2064, 2065);
2. Borra SOLO los agrupados relacionados
Primero revisa:
SQL
SELECT id, clave_documental, ruta_final
FROM documentos_agrupados
WHERE cliente_abreviatura='CIMA'
  AND anio=2026
  AND mes=1
  AND asiento_contable IN (
      SELECT asiento_contable
      FROM documentos_paginas
      WHERE id IN (2063,2064,2065)
  );
Luego elimina:
SQL
DELETE FROM documentos_agrupados
WHERE cliente_abreviatura='CIMA'
  AND anio=2026
  AND mes=1
  AND asiento_contable IN (
      SELECT asiento_contable
      FROM documentos_paginas
      WHERE id IN (2063,2064,2065)
  );
3. Borra archivos físicos distribuidos
Sí, elimina manualmente los PDFs incorrectos de:
Plain text
data/salida/2026/CIMA/01/sin_oc/
o con_oc/ si corresponde.
4. Regresa páginas a separado
SQL
UPDATE documentos_paginas
SET
    estado='separado',
    tipo_detectado=NULL,
    serie=NULL,
    numero=NULL,
    ruc_emisor=NULL,
    razon_social_emisor=NULL,
    clave_documental=NULL,
    requiere_qr=FALSE,
    qr_procesado=FALSE,
    qr_raw=NULL,
    qr_error=NULL
WHERE id IN (2063,2064,2065);
5. Reclasifica
Bash
python app/clasificar_paginas.py --year 2026 --cliente CIMA --month 1
6. Reagrupar
Bash
python app/agrupar_fragmentos.py --year 2026 --cliente CIMA --month 1
7. Redistribuir
Bash
python app/distribuir_agrupados.py --year 2026 --cliente CIMA --month 1
8. Verifica
Bash
python app/reporte_integridad_lote.py --year 2026 --cliente CIMA --month 1
